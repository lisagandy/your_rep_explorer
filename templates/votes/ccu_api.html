Congressional Close Up API

All Votes: Overview

The Congressional Close Up API identifies anomalous voters (for more information about anomalous voters please see the Individual Votes: Overview section below) and then attempts to find a reason for the anomalous behavior. The system currently examines related campaign contributions, employment data, upcoming elections, and committees that the voter might serve on. The all votes section of the api returns a list of votes that the system has examined for the specified congressional session.

Using the API:

To use the API simply specify the congressional session (currently only for 111th and 112th sessions)

Example for the 112th congress:
http://www.ccu-dev.com/votes_xml/112

Each vote is automatically assigned topics from the Jones/Baumgartner codebook. (for more information see: http://www.policyagendas.org/page/topic-codebook) However if the system does not feel confident about the assignment it will not assign any topics.

has-rel-contributions

This identifies that one or more anomalous voters have relevant campaign contribution associated with them for this particular vote.

has-rel-diff-state-senator-leanings

This identifies that one or more anomalous voters is from a state which leans in a different political direction from the voter (for example a Democrat Senator who represents a traditionally Republican stat)

has-rel-industries

This identifies that one or more anomalous voters have relevant employment (industry) data associated with them for this particular vote.

has-rel-ranking-committee

This identifies that one or more anomalous voters are a chair or vice-chair for a particular committee withthis particular vote.

has-rel-upcoming-election

This identifies that one or more anomalous voters have an upcoming election associated with them for this particular vote.

url

The url which returns more information about the particular vote, for more information please see the Individual Votes: Overview section below.

Individual Vote: Overview

The Congressional Close Up API identifies anomalous voters. In order to identify anomalous voters per vote, the system first identifies the particular bill or amendment sponsor in regards to the particular vote. It then looks at the voters from the same party as the sponsor who voted against the bill/amendment (NAY) and the voters from the opposite party who voted for the bill/amendment (AYE).

Once anomalous voters are detected then the CCU system tries to find reasons for their anomalous behavior. The system currently examines:

  • Bipartisan behavior: whether the voter tends to vote in a bipartisan manner in general
  • Contributions: whether the voter has received any related campaign contributions
  • Constituency concerns: whether there is a related industry in the voter's constituency that might be affected by the vote
  • Upcoming elections: whether there is an upcoming election that might influence the vote
  • Relevant committees: whether the voter is a member of a related committee

The CCU Individual Vote API gives a list of anomalous voters (if any) and also lists if any of the above explanations apply to each voter.

Using the API:

To use the API simply specify the congressional session, the vote number, the bill prefix and the bill number

Example for the 111th congress, Vote #156 regarding Bill S.3217:
http://www.ccu-dev.com/votes/xml=true&session=111&bill_prefix=s&bill_number=3217&vote_number=156

voter-info

Contains information and explanations concerning each anomalous voter

Parent element: voter

Child elements: None

Attributes:

lastname

Voter's lastname

firstname

Voter's firstname

osid

Unique osid assigned to the voter

gov-track-id

Unique id assigned by govtrack.org

chamber

Either Senate or House, currently only Senate as the CCU system only looks at votes that take place in the Senate

state

State the voter represents

party

Voter's party: Democrat, Republican, or other
bi-partisan-stats

Contains information and explanations concerning the with-party voting percentage of the voter, this element only appears when the particular voter is in the lowest quartile of all Senators in Congress

Parent element: voter

Child elements: None

Attributes:

party-percent-score

The percentage that the voter votes with his/her party

rank

The rank of the voter's party-percent-score among all senators

source

http://projects.washingtonpost.com
rel-contribution

This element is only present if a contribution for a particular voter from a relevant interest group is statistically significant when compared to contributions from the same interest group to other senators.

Parent element: voter

Child elements: None

Attributes:

business-type

Type of business that the contributors are associated with

business-code

Unique code of the business, associated with the business-type

total-amount

Total amount contributed from all contributors associated with the business-type

total-amt-rank-senators

How much this particular voter receives from the business compared with other senators.

amt-start-date

The starting date used to calculate total-amt

amt-end-date

The ending date used to calculate total-amt

source

http://opensecrets.org
rel-next-election

This element is only present if there is an upcoming election that might influence the particular voter.

Parent element: voter

Child elements: None

Attributes:

prediction

The prediction for the outcome of the race, either Toss Up, Lean D, Lean R, Running for Senate, Running for Governor, Running for Mayor, Potential Retirement, Potential Senate Candidate, Potential Governor Candidate, or Potential Mayoral Candidate

election-date

The date of the upcoming election

senate-class

The senate class that the voter belongs to

source-date

The date that cookpolitical.com reported the election predictions

source

http://cookpolitical.com
rel-committee

This element is only present if there is a committee that might influence the particular vote, and that the voter is a chair, vice-chair or ranking member in.

Parent element: voter

Child elements: None

Attributes:

name

Name of the committee

code

The unique code of the committee

chair

If the voter is the chair of the committee

vice-chair

If the voter is the vice-chair of the committee

ranking-member

If the voter is the ranking member of the committee
rel-industry

This element is only present if there is an industry in the voter's constituency that might be affected by the particular vote and the number of employees in that industry for the state is statistically significant when compared to other states.

Parent element: voter

Child elements: None

Attributes:

name

Name of the industry

naics-code

The unique NAICS code of the industry

state

The state the industry is located in (the same as the associated voter's state)

percent-pop-emps

Percent of the total population of the state employed by the industry

percent-pop-emps-rank-all-states

The rank of the percentage of employees for the particular state when compared to all other states

source

http://lehd.did.census.gov/led/
